### 1. **Suggesting a Resource**

The easiest way to contribute! Simply:

1. **Open an Issue** on [GitHub Issues](https://github.com/avinash201199/free-ai-agents-resources/issues)
2. **Provide the following details:**
   - Resource name and URL
   - Resource type (e.g., GitHub Repository, YouTube Playlist, Blog, Community, etc.)
   - Brief description (2-3 sentences) of what the resource offers
   - Why you think it's valuable for learning AI agents
   - Your name/GitHub handle (optional)

**Example Issue Title:** `Add: DeepSeek AI Agents Tutorial Series`

---

### 2. **Submitting a Pull Request**

Want to add resources directly to the README? Follow these steps:

#### Prerequisites
- Fork the repository to your GitHub account
- Clone your fork locally
- Create a new branch for your changes

```bash
git clone https://github.com/YOUR_USERNAME/free-ai-agents-resources.git
cd free-ai-agents-resources
git checkout -b add-awesome-resource
```

#### Guidelines for Adding Resources

**Resource Format:**
- **GitHub Repositories:** Use the provided table format with repository name as link, description (40-80 words)
- **YouTube Playlists:** Include title, link, description, and video count
- **Websites/Blogs:** Include name, URL, and clear description
- **Books/PDFs:** Include title, URL, and format (PDF/eBook)
- **Communities:** Include platform, link, and purpose

**Quality Standards:**
- âœ… Resource must be **completely free** or have a strong free tier
- âœ… Content must be **relevant to AI agents** (autonomous agents, LLM-powered systems, multi-agent frameworks)
- âœ… Resource must be **active and maintained** (no dead links, regularly updated)
- âœ… Description must be **accurate and unbiased**
- âœ… Avoid promotional/commercial resources without genuine educational value

**Example Addition (GitHub Repo):**
```markdown
| [username/awesome-agents](https://github.com/username/awesome-agents) | Comprehensive guide to building multi-agent systems with LangGraph â€” covers orchestration patterns, tool use, memory management, and evaluation frameworks. Includes 20+ working examples and deployment guides. |
```

**Example Addition (YouTube Playlist):**
```markdown
| [Building Agents with Framework X](https://www.youtube.com/playlist?list=...) | Complete video series on agent architecture, RAG integration, and production deployment â€” highly practical focus | 12 videos |
```

#### Steps to Submit PR

1. **Add your resource(s)** to the appropriate section in `README.md`
2. **Keep existing formatting** consistent with the rest of the document
3. **Verify all links work** before submitting
4. **Write a clear PR description:**
   ```
   **What:** Add [Resource Name]
   **Why:** [Brief explanation of why this resource is valuable]
   **Type:** GitHub Repository / YouTube / Blog / etc.
   **Verified:** Link tested and content reviewed
   ```
5. **Commit your changes:**
   ```bash
   git add README.md
   git commit -m "Add: [Resource Name] - brief description"
   git push origin add-awesome-resource
   ```
6. **Open a Pull Request** on the main repository with your branch
